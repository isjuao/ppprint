Welcome to ppprint!


Required data structure

For a successful upload, ppprint expects a compressed version of PredictProtein predictions. In the tar archive, there must be one directory per prediction job (== per proteome split). Each directory must contain the raw prediction files for each protein from the respective proteome split. Here, the prediction files for a single sequence must include but are not limited to files of the following extensions: .fasta, .metadisorder, .tmseg, .prona, .reprof. By the nature of PredictProtein, it is okay if certain job folders or output files are missing. For a short description of how this structure can be achieved, please read below.

Help

When running a local installation of PredictProtein, it is recommended to previously split the proteome for which prediction is to be performed into smaller .fasta files. Then, PredictProtein can be called for each .fasta file and store result files into a separate folder. That way, the user should end up with a number of job directories consisting of all prediction result files for each protein from the respective proteome split. For SARS-CoV-2, an exemplary data structure with predictions generated by PredictProtein prior to compression is provided here.
